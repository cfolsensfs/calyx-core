You are distilling a **Calyx inbox stub** (auto-generated from a git commit) into durable project knowledge.

INPUT
- The stub file (auto-*.md under .calyx/reasoning/inbox/) is attached or pasted.

GOAL
- If the commit is **noise** (trivial, formatting only): say so in one short paragraph; recommend **delete the stub**.
- If it encodes **substantive work**: produce either:
  (A) A **reasoning log** draft suitable for `.calyx/reasoning/<YYYY-MM-DD-topic>.md` (use the repo’s `_TEMPLATE` shape), OR
  (B) An **ADR stub** only if the commit implies a **binding** decision for future work.

RULES
- Do **not** invent decisions the commit did not imply.
- Link **files and commit SHA** from the stub.
- Keep **redaction**: no secrets from diffs; if stub contains sensitive paths, generalize.
- If multiple topics appear, prefer **one** reasoning log with sections or ask the human to split.
- If the thread stated **model**, **tier**, or **escalation** (see `agents/MODEL-ROUTING.md`), copy them into the optional **Agent context** section of the reasoning log or ADR — do not strip them.

OUTPUT
1. Verdict: delete stub | merge into reasoning | ADR stub needed
2. Full markdown body ready to save (or “delete only”)
